### TeamUp 기능 개요

TeamUp(팀으로 모이다)은 팀 빌딩과 매칭을 돕는 웹 애플리케이션입니다. 경로 표기는 프론트 라우팅 기준입니다.

### 인증/계정

- **로그인**: `/login`
- **내 정보/마이페이지**: `/mypage` 하위 탭

### 공모전(대회)

- **공모전 목록**: `/contests`
  - 주제/지역/마감/형식 등 기본 정보 표시
- **공모전 상세**: `/contests/:id`
  - 설명, 마감, 필요 역량 등 표시
  - **추천 사용자**: 백엔드 `/api/contests/:id/recommended-users` 기반 추천 결과 표시(스킬/지역/분야 가중)

### 팀(모집)

- **팀 목록**: `/team`
  - 필터: 지역, 분야(area), 팀 규모, 모집 상태, 마감 상태, 멤버 성향(traits)
  - 카드에서 즐겨찾기(저장) 가능
- **팀 상세**: `/team/:id`
  - 팀 기본 정보, 현재 팀원 목록 표시
  - **추천 팀원**: 백엔드 `/api/teams/:id/recommendations` 기반 추천 결과 표시
    - 하드 필터(필요 스킬/직무/시간대 등 일부), 가중치 점수(스킬/Belbin/Big Five), 이유 요약 제공
    - 데이터 부족 시 기본치와 지역/직무 보너스 적용으로 0% 편중 방지

### 프로필

- **프로필 상세**: `/profile/:id`
  - 기본 정보, 스킬, 성향(user_traits), 수상경력(awards), 포트폴리오(추가 예정)
  - **Big Five 적합도 배지 및 이유 표시**
    - 백엔드 `/api/profile/profiles/:id/fit` 호출
    - 성실성/우호성 가중, 외향성/개방성/정서안정(신경성 역가중) 반영
    - 결과: 0~10점, 레벨(매우 적합/적합/보통/부적합), 상위 이유 목록
  - **찔러보기(Nudge) 제안 버튼**: 사용자의 관심 공모전 목록을 기반으로 대상자에게 제안 전송 준비(연동됨)

### 즐겨찾기(Favorites)

- **팀/공모전 즐겨찾기**
  - 팀 상세/목록 카드에서 저장/해제 가능
  - 마이페이지에서 모아보기(연동)

### 메시지/알림

- **메시지(채팅) 테이블**: `messages` (읽음 상태 마이그레이션 스크립트 포함)
- **찔러보기(Nudges)**: `nudges` 테이블 기반 요청/응답 상태 관리

### 데이터/스키마 요약(백엔드)

- `users`: 사용자 기본 정보, 스킬, 링크(GitHub/Figma)
- `contests`: 공모전 기본 정보, 필요 역량
- `teams`, `team_members`: 팀/멤버 구성/상태
- `favorites`: 즐겨찾기(공모전/팀)
- `messages`: 채팅 메시지(1:1/팀)
- `awards`: 수상 경력
- `user_traits`: 사용자 성향(Big Five/Belbin 등 저장)
- `nudges`: 찔러보기(후킹/리마인드)

### 추천/적합도 알고리즘(요약)

- **팀원 추천(`/api/teams/:id/recommendations`)**
  - 하드 필터: 팀 미소속, 직무(옵션), 필수 스킬 보유 여부
  - 가중치 스코어: 스킬(0.35), Belbin(0.40±), Big Five(0.25∓)
  - 기본치: 스킬/Belbin/Big Five 데이터 부재 시 합리적 기본치 적용, 동일 지역/직무 보너스
- **프로필 적합도(`/api/profile/profiles/:id/fit`)**
  - Big Five 기반 점수(성실성/우호성 가중), 이유 설명 생성

### 실행 방법(요약)

- 백엔드: 포트 `4000`에서 실행(환경변수 설정 필요)
- 프론트: `http://localhost:5173` 접속
- 로그인 후 추천 API들이 활성화됨(토큰 필요)
